Using the Data Cache
<cache>
<h6 class="bg-primary text-white m-2 p-2">
Cached timestamp: @DateTime.Now.ToLongTimeString()
</h6>
</cache>



Setting Expiry
<cache expires-after="@TimeSpan.FromSeconds(15)">
<h6 class="bg-primary text-white m-2 p-2">
Cached timestamp: @DateTime.Now.ToLongTimeString()
</h6>
</cache>
I have used the expires-after attribute to specify that the content should be cached for 15 seconds.


Setting a Fixed Expiry Point
<cache expires-on="@DateTime.Parse("2100-01-01")">
<h6 class="bg-primary text-white m-2 p-2">
Cached timestamp: @DateTime.Now.ToLongTimeString()
</h6>
</cache>



Setting a Last-Used Expiry Period
<cache expires-sliding="@TimeSpan.FromSeconds(10)">
<h6 class="bg-primary text-white m-2 p-2">
Cached timestamp: @DateTime.Now.ToLongTimeString()
</h6>
</cache>

If you reload the page within 10 seconds, the cached content will be used. If you wait longer than 10 seconds
to reload the page, then the cached content will be discarded,



Using Cache Variations
<cache expires-sliding="@TimeSpan.FromSeconds(10)" vary-by-route="action">
<h6 class="bg-primary text-white m-2 p-2">
Cached timestamp: @DateTime.Now.ToLongTimeString()
</h6>
</cache>

If you use two browser tabs to request http://localhost:5000/home/index and http://localhost:5000/home/list, you
will see that each window receives its own cached content with its own expiration, since each request produces a different action
routing value.
