36.5.UnderstandingEntityFrameworkCoreContextScopeIssue
http://localhost:5000/forms/edit/2

In a Blazor application, the routing system responds to URL changes without sending new HTTP requests, which means that
multiple components are displayed using only the persistent HTTP connection that Blazor maintains to the server. This results in a
single dependency injection scope being shared by multiple components, as shown in Figure 36-7, and the changes made by one
component will affect other components even if the changes are not written to the database.

Therefore we need to manually discarding Unsaved Data Changes.
